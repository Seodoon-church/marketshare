# 특허 명세서

## 발명의 명칭

**한글**: 프랜차이즈 몰 네트워크 기반 다중 플랫폼 라이브 커머스 통합 관리 시스템 및 방법

**영문**: Franchise Mall Network Based Multi-Platform Live Commerce Integration Management System and Method

---

## 1. 기술분야

본 발명은 전자상거래 기술과 라이브 스트리밍 기술 및 프랜차이즈 유통 시스템의 융합에 관한 것으로, 더욱 상세하게는 프랜차이즈 본사가 복수의 외부 라이브 스트리밍 플랫폼에 동시에 라이브 방송을 송출하면서, 해당 방송을 프랜차이즈 네트워크 내 복수의 가맹점 온라인 스토어프론트에 자동으로 전파하고, 각 가맹점의 등급에 따른 차등 수수료를 적용하여 본사-플랫폼-가맹점 간 자동 정산을 수행하는 통합 관리 시스템 및 방법에 관한 것이다.

---

## 2. 배경기술

### 2.1 종래 기술의 현황

#### A. 라이브 커머스 시장의 성장과 한계

라이브 커머스는 실시간 영상 스트리밍을 통해 상품을 소개하고 판매하는 전자상거래 방식으로, 국내외에서 급속히 성장하고 있다. 그러나 현재의 라이브 커머스 솔루션은 다음과 같은 한계를 가진다.

첫째, 기존 라이브 커머스 플랫폼(네이버 쇼핑라이브, 카카오 쇼핑라이브, 그립(Grip) 등)은 각 플랫폼의 생태계 내에서만 동작하며, 판매자가 복수의 플랫폼에서 동시에 라이브 방송을 진행하려면 각 플랫폼에 개별적으로 방송을 설정하고 별도로 운영해야 하는 비효율이 발생한다.

둘째, 다중 플랫폼 동시 송출 기술(Restream, Streamyard 등)이 존재하나, 이는 단순한 방송 재전송에 불과하며 상품 관리, 주문 처리, 재고 동기화, 정산 등 커머스 기능이 통합되어 있지 않다.

#### B. 프랜차이즈 전자상거래의 구조적 문제

프랜차이즈 방식의 온라인 쇼핑몰 운영(분양몰)은 본사가 플랫폼을 제공하고 가맹점이 독립된 온라인 매장을 운영하는 구조로, SB플러스(sbplus.co.kr), 투비웹(tubeweb.co.kr) 등의 사업자가 존재한다. 그러나 기존 분양몰 솔루션은 다음과 같은 한계를 가진다.

첫째, 본사와 가맹점 간 상품 공유가 제한적이며, 본사 주도의 공동 마케팅 기능이 부재하다.

둘째, 본사가 라이브 방송을 진행하더라도 가맹점의 스토어프론트에 자동으로 노출되는 메커니즘이 없어, 가맹점 고객은 본사 라이브 방송의 혜택을 받지 못한다.

셋째, 라이브 방송 중 발생하는 주문에 대해 본사, 가맹점, 플랫폼 간의 수수료 정산이 수동으로 이루어져야 하며, 가맹점의 요금제(등급)에 따른 차등 수수료 적용이 자동화되어 있지 않다.

#### C. 기존 기술의 한계 종합

기존 기술을 종합하면, 라이브 커머스 기술, 다중 플랫폼 스트리밍 기술, 프랜차이즈 전자상거래 기술은 각각 독립적으로 존재하며, 이들을 유기적으로 통합한 시스템은 존재하지 않는다. 특히, 프랜차이즈 네트워크의 계층적 구조(본사-가맹점)와 다중 플랫폼 동시 라이브 커머스를 결합하고, 이를 통해 발생하는 매출에 대한 차등 수수료 자동 정산까지 일원화한 시스템은 전무하다.

### 2.2 선행기술 조사 결과

한국특허정보원(KIPRIS)의 선행기술 조사 결과, 다음과 같은 관련 특허가 확인되었으나 본 발명과는 본질적으로 상이하다.

- **라이브 커머스 관련 특허**: 단일 플랫폼 내에서의 실시간 상품 판매 시스템에 한정되며, 다중 플랫폼 동시 송출 및 프랜차이즈 네트워크 전파 기능이 없음
- **다중 스트리밍 관련 특허**: 콘텐츠 배포에 초점을 맞추고 있으며, 전자상거래 기능(상품 관리, 주문 처리, 재고 동기화, 정산) 연동이 없음
- **프랜차이즈 쇼핑몰 관련 특허**: 일반적인 전자상거래 분양몰 시스템에 한정되며, 라이브 커머스 기능이 통합되어 있지 않음

따라서 본 발명의 핵심인 "프랜차이즈 몰 네트워크 + 다중 플랫폼 동시 라이브 송출 + 가맹점 자동 전파 + 차등 수수료 자동 정산"의 조합은 선행기술이 존재하지 않는 것으로 확인되었다.

## 3. 발명이 해결하고자 하는 과제

본 발명은 상기한 종래 기술의 문제점을 해결하기 위하여 다음과 같은 과제를 해결하고자 한다.

**과제 1**: 프랜차이즈 본사가 하나의 라이브 방송 세션을 생성하면, 복수의 외부 스트리밍 플랫폼(YouTube Live, Instagram Live, TikTok Live, Facebook Live, Twitch, Naver NOW, KakaoTV, AfreecaTV 및 기타 RTMP 프로토콜 기반 플랫폼 중 선택)에 동시에 송출할 수 있는 기술적 수단을 제공하는 것

**과제 2**: 상기 송출된 라이브 방송을 프랜차이즈 네트워크 내 복수의 가맹점 온라인 스토어프론트에 자동으로 전파하여 임베드 표시함으로써, 가맹점 고객이 자신이 이용하는 가맹점 스토어프론트 내에서 본사 라이브 방송을 시청하고 상품을 구매할 수 있도록 하는 것

**과제 3**: 다중 플랫폼 및 다수 가맹점에서 동시에 발생하는 실시간 채팅 메시지를 통합 수집하여 단일 채팅 스트림으로 관리하고, 구매 알림을 실시간으로 표시하는 것

**과제 4**: 라이브 방송 중 발생하는 주문에 대해 가맹점의 요금제(등급)에 따른 차등 수수료를 자동 적용하고, 본사-플랫폼-가맹점 간 3자 자동 정산을 수행하는 것

**과제 5**: 모든 플랫폼과 모든 가맹점에서 발생하는 시청자 수, 채팅 수, 주문 수, 매출 등의 통계를 통합 대시보드에서 실시간으로 모니터링할 수 있도록 하는 것

### 4.2 각 모듈의 상세 구성 및 작용

#### 4.2.1 모듈 A: 다중 플랫폼 동시 송출 모듈

본 모듈은 프랜차이즈 본사 관리자가 생성한 하나의 라이브 방송 세션을 복수의 외부 스트리밍 플랫폼에 동시에 송출하는 기능을 수행한다.

**구성 요소:**

(a) **플랫폼 선택 인터페이스**: 본사 관리자가 9개의 지원 플랫폼 중 송출 대상 플랫폼을 선택하는 UI를 제공한다. 지원 플랫폼은 YouTube Live, Instagram Live, TikTok Live, Facebook Live, Twitch, Naver NOW, KakaoTV, AfreecaTV, 및 RTMP 프로토콜 기반 기타 플랫폼이다.

(b) **스트림 URL 관리기**: 각 선택된 플랫폼의 스트리밍 URL(임베드 URL)을 입력받아 저장하고 관리한다. 플랫폼별로 상이한 URL 형식을 자동 감지하여 적절한 임베드 코드로 변환한다.

```typescript
// 플랫폼별 임베드 URL 변환 로직
interface StreamPlatformConfig {
  platform: 'youtube' | 'instagram' | 'tiktok' | 'facebook' |
            'twitch' | 'naver' | 'kakaotv' | 'afreecatv' | 'rtmp';
  streamUrl: string;
  embedUrl: string;
  isEmbeddable: boolean;  // iframe 임베드 가능 여부
}

function convertToEmbedUrl(platform: string, rawUrl: string): string {
  switch (platform) {
    case 'youtube': {
      // https://www.youtube.com/watch?v=ABC → https://www.youtube.com/embed/ABC?autoplay=1
      const videoId = extractYouTubeId(rawUrl);
      return `https://www.youtube.com/embed/${videoId}?autoplay=1&rel=0`;
    }
    case 'twitch': {
      const channel = extractTwitchChannel(rawUrl);
      return `https://player.twitch.tv/?channel=${channel}&parent=${hostname}`;
    }
    case 'facebook': {
      return `https://www.facebook.com/plugins/video.php?href=${encodeURIComponent(rawUrl)}&autoplay=true`;
    }
    // 인스타그램, 틱톡 등 임베드 제한 플랫폼은
    // "새 탭에서 시청" 링크 + 자체 채팅 조합 방식 사용
    default:
      return rawUrl;
  }
}
```

(c) **동시 송출 상태 관리기**: 각 플랫폼의 송출 상태(예정/진행중/종료)를 독립적으로 추적하며, 전체 세션의 상태를 통합 관리한다.

#### 4.2.2 모듈 B: 프랜차이즈 네트워크 방송 전파 모듈

본 모듈은 본 발명의 가장 핵심적인 구성 요소로서, 본사의 라이브 방송을 프랜차이즈 네트워크 내 모든 가맹점의 온라인 스토어프론트에 자동으로 전파하는 기능을 수행한다.

**구성 요소:**

(a) **계층적 몰 구조 관리기**: 본사 몰(parentMall)과 가맹점 몰(childMall) 간의 계층적 관계를 관리한다. 각 몰은 고유 식별자(mallId), 부모몰 참조(parentMallId), 자식몰 목록(childMallIds)을 가진다.

```typescript
// 프랜차이즈 몰의 계층적 구조
interface Mall {
  id: string;
  name: string;
  slug: string;           // URL 경로용 고유 식별자
  ownerId: string;
  plan: PlanId;           // 'free' | 'starter' | 'business' | 'enterprise'
  commissionRate: number; // 요금제에 따른 수수료율
  parentMallId: string | null;  // 본사몰 참조 (null = 본사)
  childMallIds: string[];       // 가맹점 목록
  status: MallStatus;
  // ... 기타 필드
}
```

(b) **방송 전파 엔진**: 본사가 라이브 세션을 생성하고 'live' 상태로 전환하면, 해당 세션 정보를 모든 가맹점의 스토어프론트에 실시간으로 전파한다. 전파 메커니즘은 다음과 같다:

```typescript
// 가맹점 스토어프론트의 라이브 세션 조회 로직
function useLiveSessionsForFranchisee(mall: Mall) {
  // 1. 자체 몰의 라이브 세션 조회
  const ownSessions = useLiveSessions(mall.id);

  // 2. 본사 몰(parentMallId)의 라이브 세션 조회
  const headquartersSessions = mall.parentMallId
    ? useLiveSessions(mall.parentMallId)
    : [];

  // 3. 본사 세션은 '본사 라이브' 뱃지와 함께 가맹점 스토어프론트에 노출
  return [...headquartersSessions.map(s => ({
    ...s,
    isHeadquartersLive: true,
    displayBadge: '본사 라이브'
  })), ...ownSessions];
}
```

(c) **스토어프론트 임베드 렌더러**: 가맹점 스토어프론트에서 라이브 방송을 표시하는 방법을 제어한다. 가맹점의 메인 페이지에 "LIVE 방송중" 섹션이 자동으로 나타나며, 클릭 시 시청 페이지로 이동한다. 시청 페이지에서는 본사가 선택한 스트리밍 플랫폼(들)의 영상이 임베드되어 재생된다.

#### 4.2.3 모듈 C: 통합 실시간 채팅 관리 모듈

본 모듈은 라이브 방송 중 다중 플랫폼 및 다수 가맹점에서 발생하는 실시간 채팅을 통합 관리한다.

**구성 요소:**

(a) **Firestore 기반 실시간 채팅 엔진**: Firebase Firestore의 실시간 구독(onSnapshot) 기능을 활용하여 밀리초 단위의 실시간 채팅을 구현한다. 채팅 메시지는 `live_sessions/{sessionId}/messages` 하위 컬렉션에 저장된다.

```typescript
// 채팅 메시지 데이터 구조
interface LiveMessage {
  id: string;
  userId: string;
  userName: string;
  userProfileUrl: string | null;
  userRole: 'customer' | 'mall_owner' | 'platform_admin';
  message: string;
  type: 'chat' | 'system' | 'purchase';  // 일반채팅/시스템/구매알림
  sourceMallId: string;      // 메시지 발생 가맹점 식별
  sourceMallName: string;    // 가맹점 이름
  createdAt: Timestamp;
}
```

(b) **다중 출처 메시지 통합기**: 서로 다른 가맹점 스토어프론트에서 전송된 채팅 메시지를 하나의 채팅 스트림으로 통합한다. 본사 대시보드에서는 메시지의 출처(어느 가맹점에서 보냈는지)를 확인할 수 있다.

(c) **구매 알림 자동 생성기**: 라이브 방송 중 상품이 구매되면 "○○님이 [상품명]을 구매했습니다!" 형태의 시스템 메시지를 자동으로 생성하여 채팅 스트림에 삽입한다. 이는 다른 시청자의 구매 욕구를 자극하는 소셜 프루프(social proof) 효과를 제공한다.

#### 4.2.4 모듈 D: 라이브 상품 연동 모듈

본 모듈은 라이브 방송에 연결된 상품을 관리하고, 본사 상품을 가맹점에서 판매할 수 있도록 하는 기능을 수행한다.

**구성 요소:**

(a) **네트워크 상품 공유 시스템**: 본사가 라이브 방송에 등록한 상품은 자동으로 모든 가맹점의 라이브 시청 페이지에서 표시된다. 이는 기존 프랜차이즈 네트워크의 상품 공유 메커니즘을 활용한다.

```typescript
// 네트워크 공유 상품 구조
interface Product {
  id: string;
  name: string;
  price: number;
  salePrice: number | null;
  mallId: string;
  sourceType: 'own' | 'headquarters' | 'franchisee' | 'supplier';
  sourceMallId: string | null;     // 원본 상품의 몰
  sourceProductId: string | null;  // 원본 상품 ID
  isSharedToNetwork: boolean;      // 네트워크 공유 여부
  networkVisibility: 'all' | 'headquarters_only' | 'none';
  // ... 기타 필드
}
```

(b) **라이브 상품 캐러셀**: 시청 페이지 하단에 라이브 방송에 연결된 상품들을 가로 스크롤 형태로 표시한다. 본사 관리자는 실시간으로 하이라이트 상품(featuredProductId)을 변경할 수 있으며, 이 변경은 모든 가맹점 시청자에게 즉시 반영된다.

(c) **장바구니 연동**: 시청자가 라이브 방송 중 상품의 "장바구니 담기" 버튼을 클릭하면, 해당 시청자가 접속한 가맹점의 장바구니에 상품이 추가되고, 기존의 체크아웃 플로우를 통해 결제가 진행된다. 이때 주문에는 라이브 세션 ID가 기록되어, 라이브 방송으로 인한 매출을 추적할 수 있다.

#### 4.2.5 모듈 E: 차등 수수료 자동 정산 모듈

본 모듈은 본 발명의 비즈니스 모델을 실현하는 핵심 모듈로서, 가맹점의 요금제(등급)에 따른 차등 수수료를 자동 적용하고 정산을 수행한다.

**구성 요소:**

(a) **요금제별 차등 수수료 엔진**: 가맹점이 가입한 요금제에 따라 상이한 수수료율을 자동 적용한다.

```typescript
// 요금제별 수수료 구조
interface PricingPlan {
  id: 'free' | 'starter' | 'business' | 'enterprise';
  name: string;
  monthlyPrice: number;        // 월 고정 요금
  salesCommission: number;     // 판매 수수료율 (%)
  // 수수료율: Free(5%) > Starter(3%) > Business(1.5%) > Enterprise(0.5%)
  maxProducts: number | null;
  allowSubFranchise: boolean;  // 하위 가맹점 분양 허용
  // ... 기타 기능 제한
}

// 라이브 주문 수수료 계산
function calculateLiveOrderCommission(order: Order, mall: Mall): CommissionBreakdown {
  const plan = getPlanById(mall.plan);
  const salesAmount = order.totalAmount;

  return {
    platformCommission: salesAmount * (plan.salesCommission / 100),
    // Enterprise(0.5%) → Business(1.5%) → Starter(3%) → Free(5%)
    pgFee: salesAmount * 0.03,  // PG사 결제 수수료
    franchiseeSettlement: salesAmount - platformCommission - pgFee,
    headquartersReferral: salesAmount * (mall.referralCommissionRate / 100),
    // 본사 추천 수수료 (본사 라이브를 통한 주문인 경우)
  };
}
```

(b) **3자 자동 정산 시스템**: 라이브 방송에서 발생한 주문이 배송 완료되면, 본사(플랫폼 운영자), PG사, 가맹점 간 정산을 자동으로 수행한다.

```typescript
// 정산 데이터 구조
interface Settlement {
  id: string;
  mallId: string;           // 가맹점 ID
  mallName: string;
  period: { startDate: Date; endDate: Date };
  totalSales: number;       // 총 매출
  totalCommission: number;  // 플랫폼 수수료
  totalReferralCommission: number;  // 본사 추천 수수료
  totalSettlement: number;  // 가맹점 정산 금액
  orderCount: number;
  orderIds: string[];
  status: 'pending' | 'processing' | 'completed';
  bankInfo: BankInfo;       // 가맹점 정산 계좌
  processedAt: Date | null;
  taxInvoiceId: string | null;     // 세금계산서 연동
  settlementReportId: string | null; // 정산서
}
```

(c) **정산 보고서 자동 생성기**: 정산 기간별로 상세 보고서를 자동 생성한다. 보고서에는 주문 건별 매출, 수수료, 정산액과 함께 라이브 세션별 매출 분석이 포함된다.

#### 4.2.6 모듈 F: 크로스 플랫폼 통합 분석 모듈

본 모듈은 다중 플랫폼과 다수 가맹점에서 발생하는 모든 데이터를 통합 분석하여 본사 관리자에게 제공한다.

**구성 요소:**

(a) **실시간 통합 대시보드**: 라이브 방송 중 다음 데이터를 실시간으로 표시한다:
- 플랫폼별 동시 시청자 수 및 총 시청자 수
- 가맹점별 접속 시청자 수
- 전체 채팅 메시지 수 및 초당 메시지율
- 실시간 주문 건수 및 매출액
- 가맹점별/플랫폼별 주문 분포

(b) **세션 종료 후 분석 리포트**: 라이브 세션 종료 후 다음 분석을 제공한다:
- 최고 동시 시청자 수(peakViewerCount) 및 시간대별 추이
- 플랫폼별 주문 전환율 비교
- 가맹점별 매출 기여도
- 상품별 판매 실적
- 채팅 참여율과 구매 전환 상관관계

### 4.3 시스템의 데이터 흐름

본 발명의 시스템에서 라이브 커머스 세션의 전체 데이터 흐름은 다음과 같다.

```
[1단계: 라이브 세션 생성]
본사 관리자 → 관리 대시보드 → 라이브 세션 생성
  ├ 제목, 설명, 예약 시간 입력
  ├ 스트리밍 플랫폼 선택 (9개 중 복수 선택)
  ├ 판매 상품 선택 (네트워크 공유 상품 포함)
  └ Firestore 'live_sessions' 컬렉션에 문서 생성

[2단계: 방송 시작 및 동시 송출]
본사 관리자 → "방송 시작" 클릭
  ├ 선택된 각 플랫폼에서 개별 라이브 방송 시작
  ├ 각 플랫폼의 스트림 URL을 시스템에 등록
  ├ 세션 상태: scheduled → live
  └ Firestore onSnapshot으로 전 가맹점에 실시간 전파

[3단계: 가맹점 스토어프론트 자동 노출]
가맹점 스토어프론트 → parentMallId 기반 본사 라이브 감지
  ├ 메인 페이지: "🔴 LIVE 방송중" 섹션 자동 표시
  ├ 라이브 목록 페이지: 본사 라이브 + 자체 라이브 통합
  └ 시청 페이지: 선택된 플랫폼 영상 임베드 + 채팅 + 상품

[4단계: 시청 및 채팅]
시청자 (가맹점A 고객) → 시청 페이지 접속
  ├ 외부 플랫폼 영상 iframe 임베드 재생
  ├ Firestore 실시간 채팅 참여
  │  └ 메시지에 sourceMallId 포함 → 출처 추적
  └ 상품 캐러셀에서 상품 탐색

[5단계: 구매 및 수수료 계산]
시청자 → "장바구니 담기" → 체크아웃
  ├ 주문 생성 (liveSessionId 기록)
  ├ 가맹점 요금제 확인 → 차등 수수료 계산
  │  ├ Free(5%) / Starter(3%) / Business(1.5%) / Enterprise(0.5%)
  │  └ + PG 수수료 + 본사 추천 수수료
  ├ 결제 처리 (PortOne PG 연동)
  └ 채팅에 "○○님이 구매!" 시스템 메시지 자동 생성

[6단계: 정산]
정산 스케줄러 (Cloud Functions) → 주기적 실행
  ├ 배송 완료된 라이브 주문 집계
  ├ 가맹점별 수수료 차등 적용
  ├ 정산서 + 세금계산서 자동 생성
  └ 가맹점 등록 계좌로 정산금 지급
```

### 4.4 프랜차이즈 네트워크의 계층 구조

본 발명의 프랜차이즈 네트워크는 다음과 같은 계층 구조를 가진다.

```
[본사몰 (Enterprise)]
  parentMallId: null
  childMallIds: ['mall-a', 'mall-b', 'mall-c', ...]
  commissionRate: 0.5%
  │
  ├── [가맹점A - Business]
  │     parentMallId: '본사몰id'
  │     plan: 'business'
  │     commissionRate: 1.5%
  │     franchiseSettings: {
  │       showHeadquartersProducts: true,  ← 본사 상품 노출
  │       showNetworkProducts: true,       ← 네트워크 상품 노출
  │       customCommissionRate: null       ← 기본 수수료 적용
  │     }
  │
  ├── [가맹점B - Starter]
  │     parentMallId: '본사몰id'
  │     plan: 'starter'
  │     commissionRate: 3.0%
  │
  └── [가맹점C - Free]
        parentMallId: '본사몰id'
        plan: 'free'
        commissionRate: 5.0%
```

본사가 라이브 방송을 시작하면, `parentMallId`가 본사를 가리키는 모든 가맹점의 스토어프론트에서 해당 방송이 자동으로 감지되고 노출된다. 이때 가맹점의 `franchiseSettings.showHeadquartersProducts` 설정에 따라 본사 상품의 노출 여부가 결정된다.

### 4.5 외부 스트리밍 플랫폼 임베드 처리

본 발명은 자체 스트리밍 서버를 구축하지 않고, 외부 스트리밍 플랫폼의 임베드 기능을 활용한다. 이는 다음과 같은 기술적 이점을 제공한다.

**임베드 가능 플랫폼** (iframe 방식):
- YouTube Live: `<iframe src="https://www.youtube.com/embed/{videoId}?autoplay=1">`
- Twitch: `<iframe src="https://player.twitch.tv/?channel={channel}">`
- Facebook Live: `<iframe src="https://www.facebook.com/plugins/video.php?href={url}">`

**임베드 제한 플랫폼** (링크 + 자체 채팅 방식):
- Instagram Live, TikTok Live, Naver NOW, KakaoTV, AfreecaTV: 해당 플랫폼의 임베드 정책에 따라 "새 탭에서 시청" 링크를 제공하고, 채팅과 상품은 본 발명의 자체 시스템에서 제공

이 방식의 핵심 이점은 본사가 별도의 스트리밍 인프라를 구축할 필요 없이, 이미 수십억 사용자를 보유한 외부 플랫폼의 인프라를 활용하면서도, 커머스 기능(채팅, 상품, 주문, 정산)은 자체 시스템에서 통합 관리할 수 있다는 것이다.

## 5. 발명의 효과

### 5.1 기술적 효과

#### A. 프랜차이즈 네트워크 시너지 극대화
- 본사의 하나의 라이브 방송이 N개 가맹점에 동시 노출되어 도달 범위가 N배 확대
- 가맹점은 별도의 방송 장비나 인력 없이 본사의 라이브 커머스 인프라를 활용 가능
- 가맹점 고객은 자신이 이용하는 가맹점 스토어프론트 내에서 일관된 쇼핑 경험 유지

#### B. 다중 플랫폼 동시 도달
- 9개 스트리밍 플랫폼 중 선택하여 동시 송출으로 최대 시청자 도달 범위 확보
- 플랫폼별 사용자 특성에 따른 다양한 고객층 확보
- 하나의 방송 세션으로 복수 플랫폼 운영에 따른 운영 비용 절감

#### C. 차등 수수료 자동 정산
- 가맹점 등급에 따른 수수료율 자동 적용으로 정산 오류 제거
- 본사-PG사-가맹점 간 3자 정산의 완전 자동화
- 정산서 및 세금계산서 자동 생성으로 세무 처리 간소화

#### D. 실시간 통합 분석
- 다중 플랫폼 + 다수 가맹점의 데이터를 단일 대시보드에서 실시간 모니터링
- 라이브 방송의 ROI(투자 대비 수익)를 정확히 측정 가능
- 데이터 기반의 라이브 커머스 전략 수립 지원

### 5.2 산업적 효과

#### A. 프랜차이즈 커머스 시장 혁신
- 기존 오프라인 중심의 프랜차이즈 모델에 라이브 커머스를 결합한 새로운 비즈니스 모델 창출
- 프랜차이즈 본사의 마케팅 효율성을 획기적으로 향상
- 가맹점의 온라인 매출 확대를 위한 신규 채널 제공

#### B. 중소 가맹점의 라이브 커머스 진입 장벽 해소
- 개별적으로 라이브 커머스를 운영하기 어려운 소규모 가맹점도 본사 라이브 방송을 통해 라이브 커머스의 혜택을 누릴 수 있음
- 방송 장비, 인력, 기술적 노하우 없이도 라이브 커머스 매출 창출 가능

#### C. 글로벌 확장성
- 외부 스트리밍 플랫폼의 글로벌 인프라를 활용하므로, 해외 시장 진출 시 별도의 스트리밍 인프라 구축 불필요
- 프랜차이즈 모델의 특성상 해외 가맹점 추가 시 동일한 시스템으로 운영 가능

## 6. 도면의 간단한 설명

### 도면 1: 전체 시스템 아키텍처도
프랜차이즈 몰 네트워크 기반 다중 플랫폼 라이브 커머스 통합 관리 시스템의 전체 구성을 나타내는 블록도. 본사 관리 대시보드, 다중 플랫폼 동시 송출 모듈, 프랜차이즈 네트워크 방송 전파 모듈, 통합 채팅 관리 모듈, 라이브 상품 연동 모듈, 차등 수수료 정산 모듈, 통합 분석 모듈 간의 연결 관계를 도시한다.

### 도면 2: 다중 플랫폼 동시 송출 프로세스
본사 관리자가 라이브 세션을 생성하고 복수의 외부 스트리밍 플랫폼에 동시 송출하는 과정을 나타내는 흐름도. 9개 플랫폼(YouTube Live, Instagram Live, TikTok Live, Facebook Live, Twitch, Naver NOW, KakaoTV, AfreecaTV, RTMP)의 선택, URL 등록, 상태 관리 과정을 도시한다.

### 도면 3: 프랜차이즈 네트워크 방송 전파 메커니즘
본사의 라이브 방송이 N개 가맹점의 스토어프론트에 자동 전파되는 과정을 나타내는 시퀀스 다이어그램. parentMallId 기반 계층 구조, 실시간 상태 전파(Firestore onSnapshot), 가맹점 스토어프론트의 자동 노출 과정을 도시한다.

### 도면 4: 차등 수수료 자동 정산 프로세스
가맹점 등급(Free/Starter/Business/Enterprise)에 따른 차등 수수료 계산 및 3자 정산(본사-PG사-가맹점) 과정을 나타내는 흐름도. 주문 발생부터 정산 완료까지의 전 과정을 도시한다.

### 도면 5: 통합 실시간 채팅 및 구매 알림 시스템
다중 가맹점에서 발생하는 채팅 메시지가 Firestore를 통해 통합되고, 구매 시 자동 알림이 생성되는 과정을 나타내는 데이터 흐름도.

### 도면 6: 라이브 커머스 시청 페이지 구성도
시청자가 보게 되는 라이브 커머스 시청 페이지의 레이아웃. 상단 제목 바(LIVE 뱃지, 제목, 시청자/채팅 카운트), 중앙 그리드(영상 플레이어 + 실시간 채팅), 하단 상품 섹션(하이라이트 상품 + 상품 그리드)의 구성을 도시한다.

### 도면 7: 라이브 세션 데이터 모델 및 Firestore 구조
라이브 세션, 채팅 메시지, 주문, 정산 간의 데이터 관계를 나타내는 ERD(Entity Relationship Diagram). Firestore 컬렉션 구조(live_sessions, messages 하위 컬렉션, orders, settlements)를 도시한다.

## 7. 실시예

### 실시예 1: 뷰티 프랜차이즈의 본사 라이브 쇼핑

**상황**: 뷰티 프랜차이즈 본사가 신제품 런칭 라이브 쇼핑을 진행하며, 전국 50개 가맹점의 스토어프론트에 동시 노출한다.

**적용 플랫폼**: YouTube Live, Instagram Live, TikTok Live (3개 동시 송출)

**가맹점 구성**:
- Enterprise 가맹점 5개 (수수료 0.5%)
- Business 가맹점 15개 (수수료 1.5%)
- Starter 가맹점 20개 (수수료 3%)
- Free 가맹점 10개 (수수료 5%)

**동작 과정**:
1. 본사 관리자가 관리 대시보드(`/mall-admin/live/create`)에서 라이브 세션 생성
   - 제목: "신상 봄 컬렉션 라이브"
   - 플랫폼: YouTube, Instagram, TikTok 선택
   - 상품: 봄 신상 화장품 15종 선택 (isSharedToNetwork: true)
2. 예약 시간에 YouTube, Instagram, TikTok에서 각각 라이브 방송 시작
3. 각 플랫폼의 스트림 URL을 시스템에 등록 → 세션 상태: 'live'
4. 50개 가맹점의 스토어프론트에서 자동으로 "🔴 LIVE 방송중" 섹션 표시
5. 가맹점 고객이 자신의 가맹점 페이지에서 라이브 시청
   - YouTube 임베드로 영상 재생
   - Firestore 실시간 채팅 참여
   - 하단 상품 캐러셀에서 상품 탐색
6. 고객이 상품 구매 시:
   - 해당 가맹점의 장바구니에 추가
   - 가맹점 등급에 따른 수수료 자동 적용
   - 채팅에 "○○님이 봄 에센스를 구매했습니다!" 표시
7. 방송 종료 후:
   - 50개 가맹점별 매출/주문 현황 통합 리포트 생성
   - 등급별 차등 수수료 적용된 정산서 자동 생성

**결과**:
- 본사 1회 방송 → 50개 가맹점 × 3개 플랫폼 = 최대 150개 접점 동시 노출
- 가맹점별 별도 방송 불필요 → 인력/비용 50배 절감
- 등급별 차등 수수료 자동 정산 → 수작업 정산 오류 제거

### 실시예 2: 식품 프랜차이즈의 지역 특산물 라이브 판매

**상황**: 지역 식품 프랜차이즈가 산지 직송 특산물 라이브 판매를 진행한다.

**적용 플랫폼**: YouTube Live, Naver NOW (2개 동시 송출)

**특징적 기능 활용**:
- 본사가 산지에서 직접 방송 → 2개 플랫폼 동시 송출
- 30개 가맹점 스토어프론트에 자동 전파
- 각 가맹점의 배송 설정(ShippingZone)에 따라 배송비 자동 계산
- 가맹점별 포인트 적립률(PointSettings) 적용
- 회원 등급별(MemberGrade) 추가 할인 자동 적용

**수수료 정산 예시** (1건 10만원 주문):
| 가맹점 등급 | 수수료율 | 플랫폼 수수료 | PG 수수료(3%) | 가맹점 정산 |
|------------|---------|-------------|-------------|-----------|
| Enterprise | 0.5% | 500원 | 3,000원 | 96,500원 |
| Business | 1.5% | 1,500원 | 3,000원 | 95,500원 |
| Starter | 3.0% | 3,000원 | 3,000원 | 94,000원 |
| Free | 5.0% | 5,000원 | 3,000원 | 92,000원 |

### 실시예 3: 패션 프랜차이즈의 글로벌 라이브 쇼핑

**상황**: 패션 프랜차이즈가 해외 가맹점을 포함한 글로벌 라이브 쇼핑을 진행한다.

**적용 플랫폼**: YouTube Live, Instagram Live, TikTok Live, Facebook Live, Twitch (5개 동시 송출)

**글로벌 확장 시나리오**:
- 국내 가맹점 100개 + 일본 가맹점 20개 + 미국 가맹점 10개
- 5개 플랫폼 동시 송출 → 130개 가맹점 자동 전파
- 각 국가의 결제 시스템(PG)과 연동
- 본사 1회 방송으로 글로벌 130개 접점에서 동시 판매
- 가맹점별 요금제에 따른 수수료 자동 정산

## 8. 기술적 우수성 및 산업상 이용가능성

### 8.1 기존 기술 대비 우수성

#### A. 기존 기술과의 차별점

| 비교 항목 | 기존 라이브 커머스 | 기존 다중 스트리밍 | 기존 분양몰 | **본 발명** |
|----------|------------------|------------------|-----------|------------|
| 다중 플랫폼 동시 송출 | X (단일) | O | X | **O (9개)** |
| 프랜차이즈 네트워크 | X | X | O | **O** |
| 본사→가맹점 방송 전파 | X | X | X | **O (자동)** |
| 실시간 채팅 통합 | 단일 플랫폼 | X | X | **O (다중 출처)** |
| 차등 수수료 정산 | X | X | 수동 | **O (자동)** |
| 상품 네트워크 공유 | X | X | 제한적 | **O (완전)** |

#### B. 핵심 차별화 기술

1. **3단 구조(Triple-Layer Architecture)**: 다중 플랫폼 → 프랜차이즈 전파 → 차등 정산의 유기적 3단 구조는 기존 어떤 시스템에서도 구현되지 않은 본 발명 고유의 아키텍처이다.

2. **Firestore onSnapshot 기반 실시간 전파**: Firebase Firestore의 실시간 데이터 구독 기능을 활용하여, 본사의 라이브 세션 상태 변경이 밀리초 단위로 모든 가맹점에 전파되는 메커니즘은 별도의 푸시 서버나 WebSocket 서버 구축 없이 구현 가능하다.

3. **외부 플랫폼 임베드 + 자체 커머스 하이브리드**: 영상은 외부 플랫폼(YouTube 등)의 인프라를 활용하면서, 채팅/상품/주문/정산은 자체 시스템에서 처리하는 하이브리드 아키텍처는 인프라 비용을 최소화하면서 커머스 기능의 완전한 통제를 가능하게 한다.

### 8.2 산업상 이용가능성

#### A. 즉시 적용 가능한 산업 분야

1. **뷰티/화장품 프랜차이즈**: 본사 주도 신제품 라이브 런칭
2. **식품/농산물 프랜차이즈**: 산지 직송 라이브 판매
3. **패션/의류 프랜차이즈**: 시즌 컬렉션 라이브 쇼핑
4. **건강/의료기기 프랜차이즈**: 전문가 소개 라이브 판매
5. **교육/도서 프랜차이즈**: 저자 인터뷰 + 도서 판매
6. **가전/전자 프랜차이즈**: 제품 시연 라이브 판매

#### B. 기대되는 비즈니스 모델

1. **SaaS 구독형**: 프랜차이즈 본사에게 월 정기 구독으로 제공
2. **거래 수수료형**: 라이브 방송 매출의 일정 비율을 플랫폼 수수료로 수취
3. **하이브리드**: 기본 구독료 + 매출 수수료의 결합
4. **White-label 라이선스**: 대기업/유통사에게 자체 브랜드로 제공
