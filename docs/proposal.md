## 1. 제안 개요

귀사의 분양몰 솔루션 개발 요청에 대해, 검증된 자체 프레임워크를 기반으로 한 개발 제안을 드립니다.

저희는 이미 **MarketShare**라는 분양몰 전용 플랫폼을 자체 개발하여 운영 중이며, 이 과정에서 축적한 분양몰 도메인 지식과 기술력을 바탕으로 귀사 맞춤형 솔루션을 제공할 수 있습니다.

### 왜 저희인가?

| 강점 | 설명 |
|------|------|
| **이미 만들어본 경험** | 분양몰 플랫폼을 직접 설계/개발/배포한 실전 경험 보유 |
| **검증된 코드베이스** | 80개 페이지, 21개 서비스, 13개 자동화 함수 — 이미 작동하는 시스템 |
| **분양몰 도메인 전문** | SB플러스, 투비웹 등 기존 솔루션을 분석하여 핵심 기능 파악 완료 |
| **최신 기술 스택** | Next.js 16 + Firebase + TypeScript — 유지보수 용이, 확장성 우수 |
| **빠른 개발** | 핵심 모듈이 이미 구현되어 있어 기존 솔루션 대비 개발 기간 대폭 단축 |

## 2. 분양몰 솔루션이란?

분양몰 솔루션은 **하나의 플랫폼에서 다수의 독립 쇼핑몰을 생성·운영·관리**할 수 있는 시스템입니다.

```
┌─────────────────────────────────────────────────┐
│              귀사 메인 플랫폼                      │
│        (통합 관리 / 정산 / 통계 / 회원)             │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ 분양몰 A  │  │ 분양몰 B  │  │ 분양몰 C  │ ... │
│  │ shop-a.kr │  │ shop-b.kr │  │ shop-c.kr │     │
│  └──────────┘  └──────────┘  └──────────┘      │
│       │                            │            │
│  ┌────┴────┐                 ┌────┴────┐       │
│  │하위몰 A-1│                 │공급사 X  │       │
│  │하위몰 A-2│                 │공급사 Y  │       │
│  └─────────┘                 └─────────┘       │
│                                                 │
└─────────────────────────────────────────────────┘
```

**일반적인 활용 사례:**
- 본사가 분양몰을 개설해주고 월 이용료 + 판매 수수료로 수익 창출
- 프랜차이즈 본사가 가맹점별 온라인 쇼핑몰 제공
- 도매 업체가 소매 판매자에게 상품 + 쇼핑몰을 함께 공급
- 교육/서비스 업체가 지역별 독립 사이트 운영

### 3.4 자동화 (서버 관리 불필요)

| 자동화 항목 | 동작 |
|------------|------|
| 상품 등록 → 메인마켓 동기화 | 분양몰 상품이 자동으로 통합 마켓에 노출 |
| 주문 생성 → 재고 차감 + 통계 갱신 | 주문 즉시 자동 처리 |
| 배송 완료 → 포인트 자동 적립 | 구매 확정 시 설정된 비율로 적립 |
| 주문 취소 → 포인트/쿠폰 자동 복구 | 사용된 포인트/쿠폰 원복 |
| 정산 완료 → 정산서 + 세금계산서 자동 생성 | 수작업 없이 자동 발행 |
| 매일 자동 → 만료 포인트/쿠폰 처리 | 기한 경과 항목 자동 비활성화 |
| 매일 자동 → 회원등급 재평가 | 구매 금액 기반 등급 자동 조정 |

## 4. 기술 스택

| 영역 | 기술 | 이점 |
|------|------|------|
| **프론트엔드** | Next.js 16 + React 19 | SSG 정적 빌드, SEO 최적화, 빠른 로딩 |
| **스타일링** | Tailwind CSS v4 | 반응형 디자인, 일관된 UI, 빠른 커스텀 |
| **백엔드** | Firebase (Firestore, Auth, Storage) | 서버리스, 자동 스케일링, 실시간 DB |
| **자동화** | Firebase Cloud Functions | 서버 관리 불필요, 이벤트 기반 처리 |
| **결제** | PortOne (아임포트) | 5개 PG사 통합, 안정적 결제 |
| **언어** | TypeScript (100%) | 타입 안전, 버그 최소화, 유지보수 용이 |
| **배포** | Firebase Hosting | CDN, SSL 자동, 커스텀 도메인 |

### 이 스택을 선택한 이유

| 비교 | 전통 방식 (PHP+MySQL) | MarketShare 방식 |
|------|---------------------|-----------------|
| 서버 | 별도 호스팅 필요 (월 5~30만 원) | **서버리스** (사용량 기반, 초기 무료) |
| DB | MySQL 관리/백업 필요 | **Firestore** 자동 백업/스케일링 |
| 보안 | 직접 SSL, 방화벽 설정 | **자동 SSL**, Firebase 보안 규칙 |
| 스케일링 | 트래픽 증가 시 서버 증설 | **자동 스케일링** |
| 유지보수 | 서버 패치, DB 튜닝 필요 | **관리 비용 거의 없음** |

## 5. 개발 일정

저희는 핵심 모듈이 이미 개발 완료되어 있어, **귀사 맞춤 커스터마이징**에 집중할 수 있습니다.

### 개발 단계

| 단계 | 기간 | 내용 |
|------|------|------|
| **1단계: 기획/설계** | 1주 | 요구사항 확인, 화면 설계, DB 구조 확정, 디자인 방향 협의 |
| **2단계: 핵심 기능** | 2주 | 플랫폼 관리자, 분양몰 관리자, 고객 쇼핑 기능, 결제 연동 |
| **3단계: 부가 기능** | 1주 | 포인트/쿠폰/등급, 배송 관리, 엑셀 처리, 공급사 포탈 |
| **4단계: 정산/자동화** | 1주 | 정산 시스템, 세금계산서, Cloud Functions, 알림 |
| **5단계: 디자인/QA** | 1주 | 귀사 브랜드 적용, UI 커스텀, 테스트, 버그 수정 |
| **6단계: 배포/인수** | 1주 | 프로덕션 배포, 운영 가이드, 인수인계 |

> **총 개발 기간: 약 7주**
>
> 동일 규모의 솔루션을 처음부터 개발할 경우 4~6개월이 소요되지만, 이미 검증된 코드베이스를 활용하여 **7주**로 단축합니다.
